Card icon coloring

The card's existing 40×40 icon tile, tinted to coordinate with the first label's color. The icon does the color signaling so the labels stay quiet (/text). Same four label colors (blue / green / purple / red) shown across each variant.

A · duotone icon · tile bg stays neutral cream
B · duotone icon · tile bg = label-bg (color wash)
C · duotone icon · cream tile · border in label color
D · fill weight icon · solid tile (most saturated)
E · variant B but mixed with no-label fallback

Read:

  • A — quietest. Only the icon picks up label color; tile bg stays cream. Subtle but readable. Best for restraint.
  • B — tile bg becomes a color wash matching the label. Strong color signal at one glance, still calm. This is the most "Notion-y" feel.
  • C — neutral tile but border in label color. Distinct silhouette but slightly louder than A.
  • D — solid label bg + filled icon weight. Maximum saturation. Risks looking like a sticker on the card.
  • E — variant B in a real grid showing the no-label fallback (cream tile, muted icon) so you can see how mixed boards read.

My pick: B — color wash on the tile + duotone icon in the matching tone. The icon does dual duty (decoration + color tag), labels stay quiet (/text), and the no-label fallback (E middle card) preserves the original card chrome. Wiring it would mean: in Card.jsx, derive a "primaryColor" from labels[0]?.color, and conditionally swap the tile's bg + icon color tokens. ~6 lines.
