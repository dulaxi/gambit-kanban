Confirmation popups — current state

Every destructive confirmation in Kolumn at production size, faithful to the live render. Five share one component (ConfirmModal); one is an inline panel (Settings → wipe data). Use this as the visual reference when redesigning the chrome — change ConfirmModal once and all five modals update.

01 Delete board

Delete board

This will permanently delete the board and all its tasks.

Triggered from: Sidebar → board row → Trash icon Source: Sidebar.jsx:346 Destructive scope: board + all its columns + all cards Reversibility: 5s undo toast after dismiss
02 Leave board

Leave board

You'll lose access to this board. The owner can re-invite you later.

Triggered from: Sidebar / WorkspaceSidebar / BoardShareModal (three sites) Source: Sidebar.jsx:358 / WorkspaceSidebar.jsx:210 / BoardShareModal.jsx:340 Destructive scope: revokes user's membership; data preserved Reversibility: no — owner must re-invite
03 Delete column

Delete "In Progress"

This section has 5 task(s) that will be permanently deleted.

Triggered from: Board → column header menu → Delete column Source: Column.jsx:272 Destructive scope: column + its cards Reversibility: 5s undo toast Note: title interpolates column name; message interpolates card count
04 Delete workspace (owner)

Delete workspace

This permanently removes the workspace and unlinks its boards. This cannot be undone.

Triggered from: Workspace settings → Danger zone → Delete workspace Source: WorkspaceDangerZone.jsx Destructive scope: workspace metadata; boards become personal (unlinked) Reversibility: no — boards keep their data but lose workspace
05 Leave workspace (member)

Leave workspace

You'll lose access to all boards in this workspace.

Triggered from: Workspace settings → Danger zone → Leave workspace Source: WorkspaceDangerZone.jsx Destructive scope: revokes user's workspace membership Reversibility: no — owner must re-invite
06 Wipe all data (inline panel, not a modal)

Confirm — this cannot be undone

Permanently delete all your boards, notes, and settings.

Triggered from: Settings page → Account → Clear all data Source: SettingsPage.jsx Destructive scope: all boards + notes + settings (full account wipe) Reversibility: no Note: the only confirmation that is NOT a modal — it's an inline expanded panel

Redesign axes — questions to answer

Tone: copper (warning) vs neutral (matter-of-fact) vs full destructive red?
Icon: warning triangle, trash, or none? Phosphor weight (regular / fill / duotone)?
Geometry: current is icon-left / title-right / message-below-with-indent. Alternatives: icon-top-centered (more dramatic), icon-inline-with-message (compact), no-icon-large-title (most stripped).
Button placement: right-aligned (current macOS), full-width split-pair, ghost-cancel-on-left + destructive-right.
Severity differentiation: currently every confirm looks identical regardless of how destructive it is. "Delete one column" gets the same chrome as "Delete entire workspace". A redesign could introduce a severity axis (e.g., type-to-confirm for catastrophic actions).
Settings inline panel (#6) consistency: reshape it to match the modal chrome, or keep it deliberately different to signal "you opened this drawer yourself, no surprise modal."
