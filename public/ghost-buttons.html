Ghost buttons — what & where

A "ghost" button is one that has no visible chrome at rest (transparent background, no border) and only reveals itself on hover. It carries the same height/padding/font as a primary button — so vertical rhythm is preserved when ghost & primary sit next to each other — but it visually recedes until interacted with. Used for tertiary / supportive actions where calling attention would compete with the primary intent.

When to reach for ghost

Three load-bearing rules:

  • Pair with a primary action. The ghost is the "less attention" half of a Cancel / Save pair. If a button can stand alone, it should usually be secondary, not ghost.
  • Use for icon-only utility buttons in toolbars or composer chrome — Add files, Voice mode, Close, Toggle menu, etc. The transparent rest state stops them from competing with the input/textarea they sit next to.
  • Don't use ghost for the main action. Ghost says "I'm here if you need me" — never make a Save, Submit, Confirm, or Generate button ghost. Use primary or accent.

Pattern recap

If you're about to add a button, this is the rule of thumb:

  • Primary action that creates / saves / confirms: variant="primary" (default) or variant="accent" for AI-positive actions.
  • Destructive (delete / discard / sign out): variant="destructive".
  • The "less attention" half of a pair (Cancel, Skip, Dismiss): variant="ghost".
  • An icon-only utility action in a toolbar or chrome: variant="ghost" size="icon-sm".
  • A standalone subdued action that's not paired with anything (rare): variant="secondary" — has a faint sand border so it's findable on a blank surface.
