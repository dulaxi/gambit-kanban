Coherency takeaways

What a future polish-phase pass should resolve, ranked by impact.

1. Pick one selected-state pattern

Right now: 4 different visual treatments for "this is selected." Recommend Pattern B (lime-wash + 1px lime ring) as canonical — works equally well for cards, templates, icons, pills. Migrate Card to use ring instead of border.

2. Resolve notification-color split

Lime = "new / positive thing." Copper = "needs urgent action." Currently the rule isn't enforced — one count badge is lime, another is copper. Document the split in CLAUDE.md and migrate.

3. Migrate hardcoded hex to tokens

11+ raw hex codes still in components (#C27A4A, #D4A843, #8B7355, etc.). All have token equivalents in :root. Should be a mechanical search-and-replace.

4. Fill the search-match gap

SearchDialog doesn't highlight matched query terms. Use the same lime-wash-text pattern as @mentions for inline-emphasis consistency.

5. Audit button focus visibility

Most buttons don't define :focus-visible at all — keyboard users see no ring on Tab. Card got it right. Bake it into the future Button primitive.

6. Drop the lime focus ring on inputs

CreateBoardModal + LandingPage still use focus:ring-2 ring-[#C2D64A]. CLAUDE.md says 1px ink border. Two-line fix in each file.
