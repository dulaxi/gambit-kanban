DECISION 1

Destructive color: bark or copper?

Code currently uses bark (a brown taupe). The mockup says copper (warmer, orange-leaning). They read very differently — bark feels like "permanent / heavy"; copper feels like "warning / hot." Pick one and we standardize.

DECISION 2

Default size: 32px or 36px tall?

Both exist in the codebase. 32px (h-8) is denser, common in toolbars and tight UIs. 36px (h-9) is roomier, easier touch target. The default is the one you'll use most — picking the right one matters.

A · 32px (h-8) · denser
height: 32px · padding: 0 12px · font: 12px
B · 36px (h-9) · roomier
height: 36px · padding: 0 14px · font: 13px

The codebase mixes rounded-lg (8px) and rounded-xl (12px). 8px reads sharper / more pro; 12px reads softer / more friendly. Need to pick one for buttons.

API · CODE-ONLY

Two non-visual decisions

These don't change how the button looks, only how it's written. Skim and tell me your preference.

Polymorphism

A · as prop — <Button as="a" href="/x"> renders as a link styled as a button. Lets NavLinks / external links use Button styles without wrapping.

B · plain <button> only — link-styled buttons require a separate component or wrapping a NavLink in a span. Simpler, less magic.

Icon composition

A · props — <Button leftIcon={'<'}Plus />}>New board</Button>. Type-safe, easy to spot what's an icon. Slight rigidity.

B · children — <Button><Plus /> New board</Button>. Pure composition; no special slots. Slightly noisier but flexible.
